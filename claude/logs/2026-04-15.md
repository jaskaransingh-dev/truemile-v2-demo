## What's broken / in progress
- **Driver.driverId is always null** in dispatcher response — the Prisma Driver model uses cuid string `id` and has no numeric driverId field. Needs schema change if numeric IDs are required.
- **Dispatcher seed fallback only covers 4 names** (Max/John/Paul/Mike). Any other driver in the DB returns empty location/emptyTime fields. Needs a real-data pipeline before adding more drivers.
- **Background location in Expo Go is unreliable on iOS** — 15-min interval pings may not fire. Manual ping button works for testing. Full background requires EAS build.
- **Dispatcher login requires pre-created Supabase users** — no signup UI. Must create dispatcher accounts via Supabase Auth dashboard.
- **POD/BOL email attachment name** uses `.jpg` extension even though we store scanned buffer — harmless but cosmetically wrong if the source was a PDF. Not currently an issue since mobile only sends JPEGs.
- **No auth on dispatcher API endpoints** — `/api/dispatcher/drivers` is open. Fine for now since it's dev-mode only but needs tenantScope middleware for production.
- **`TEST ONLY, remove before production`** comments on manual ping button in HomeScreen — cleanup task before launch.

## Decisions made
- **Phone format is 11-digit with leading "1", no plus**: e.g. `14698473017`. Normalized at every read/write point for defense-in-depth. `+1` and formatted variants never hit the backend.
- **Gmail API for email with attachments, not nodemailer**: Reused GmailRuntimeService OAuth pattern. Built multipart/mixed MIME manually since the existing service only supported plain text. Avoids adding a new dep.
- **Sharp-server-side scan processing, not mobile**: Mobile does resize + JPEG compression only. True grayscale/contrast happens in the backend before emailing. Keeps mobile bundle light and results are consistent across iOS/Android.
- **Scan "look" on mobile is UX-only**: White frame + green corner markers + scan animation. No native image filters — those would require a custom EAS build.
- **Seed fallback in dispatcher endpoint**: Rather than forcing real DriverOperationalState rows for all drivers, fall back to hardcoded demo data keyed by driver name. Acceptable for demo; needs replacement for production.
- **Dispatcher map auto-refreshes every 60s AND subscribes to Realtime**: Belt-and-suspenders. Realtime gives instant updates when a driver pings; 60s refresh catches anything missed.
