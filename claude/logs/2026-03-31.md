# 2026-03-31

## What was completed

- **content.js — settings cache refactor**: Removed `chrome.storage.local.get` from inside `runRanker`. Added module-level `cachedSettings` with Royal Carriers defaults (REEFER, 1.62/1.86 RPM, 1.606 CPM, 650 mi/day, 11/2 cycle). Storage loaded once at inject time (non-blocking). `runRanker` now reads `const s = cachedSettings` directly — no async callback.
- **content.js — scraper selector fixes (3 bugs)**:
  - Miles: `[data-test="trip-miles"]` → `[data-test="load-trip-cell"]`
  - Deadhead: old `[data-test="deadhead"]` → `[data-test="load-dho-cell"] .deadhead` with paren-stripping (`(32)` → `32`)
  - Origin/destination: replaced `parseCityState()` fallback chain with direct DOM reads from `[data-test="load-origin-cell"]` and `[data-test="load-destination-cell"]` using `.city-state-container:first-child .truncate` / `.state` selectors confirmed against live DAT DOM
- **content.js — `parseCityState` removed**: No longer needed; city and state read from separate nodes.
- **content.js — `description: 'team load'` on RM loads**: Spread into `results.push()` so engine can detect and reject team loads.
- **content.js — click-to-scroll + flash**: Ranked and rejected sidebar cards now scroll the matching DAT row into view and flash a yellow outline for 1.5s on click.
- **content.js — simulate DAT row click**: After scroll settles (300ms delay), the DAT row receives a programmatic `.click()` to open the detail panel.
- **content.js — `getClickableRow` helper**: Walks up from `rowEl` to find the nearest ancestor with class containing `row-cells` — the actual clickable container in DAT's Angular DOM.
- **content.js — `currentlyOpenRow` tracker**: Module-level variable tracks which DAT row's detail panel is open. Before opening a new panel, the previous row is clicked to close it (150ms wait for animation).
- **content.js — card handlers made `async`**: Required for `await` on close-animation delay.

## What's broken / in progress

- **`currentlyOpenRow` stale reference**: If DAT closes the detail panel autonomously (Escape, click-elsewhere), `currentlyOpenRow` still holds the old row. Next sidebar card click will toggle the old panel open before opening the new one. Fix requires knowing DAT's selected-row class name or adding a MutationObserver on the panel container. Decision deferred.
- **Shadow testing**: Not yet done — need to run real Royal Carriers loads through the extension against the backend and record actual vs engine decisions. This is the primary Phase 1 deliverable.

## Decisions made

- **Default driver profile in `cachedSettings` is Royal Carriers spec**: REEFER, minRPM 1.62, targetRPM 1.86, variableCPM 1.606, 650 mi/day, 11-day cycle, 2 home days. These are the live operator values, not generic defaults.
- **`parseCityState` removed entirely**: Origin/destination city and state are now read from separate DOM nodes. No string parsing needed.
- **`getClickableRow` walks DOM at click time**: Not cached at scrape time. DAT's Angular may re-render rows between scrape and click; walking at click time is more reliable.
