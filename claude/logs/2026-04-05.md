# 2026-04-05 / 2026-04-06

## What was completed

- **Scoring overhaul in decision-engine-v2.ts**: Replaced 5-dimension weighted formula with 2-dimension: `score = revenuePerDayScore × 0.60 + rpmScore × 0.40`. MCI, cycleFit, deliveryScore, netProfitScore still computed and returned for display but no longer factor into base score.
- **MCI multiplier**: Range-based post-multiplier applied to baseScore. `finalScore = min(baseScore × mciMultiplier, 1.0)`. Values: ≥50→1.10, ≥10→1.05, ≥-10→1.00, ≥-50→0.90, <-50→0.80. Loose markets drop score by 10%, Very Loose by 20%.
- **estimatedDays step function**: `<600mi=1d, 600–1269=2d, 1270–1799=3d, 1800–2399=4d, ≥2400=5d`.
- **rpmScore**: `min(effectiveRPM / targetRPM, 2.0) / 2.0`. Uses loaded miles only.
- **revenuePerDayScore**: relative normalization — best load in set = 1.0.
- **normalizeTrailerType()**: Exported helper in decision-engine-v2.ts. Maps VAN/DRYVAN/V→DRY_VAN, REEFER/R→REEFER, FLATBED/FLAT/F→FLATBED, FLEX/VR→FLEX. Applied at route handler and engine entry. Logs warning for unrecognized values.

## Decisions made

- **2-dimension scoring + MCI multiplier**: baseScore = rpdScore×0.60 + rpmScore×0.40, then × mciMultiplier. MCI affects ranking meaningfully (±10-20%) without dominating.
- **Popup setting is trailer type source of truth**: Auto-detect from DAT filter is fallback only (used when no saved setting exists). Prevents silent overwrites.
- **normalizeTrailerType at both layers**: Route handler normalizes before building objects, engine normalizes again at entry. Belt-and-suspenders.
- **FLEX loads never rejected for trailer mismatch**: Simplified from REEFER/DRY_VAN-only acceptance to universal acceptance.
- **estimatedDays step function**: Hard brackets match dispatcher mental model better than continuous division.
- **effectiveRPM = rate / miles**: Loaded miles only. Deadhead from scraper is unreliable.
- **geocodeCity as MCI proxy fallback**: Cached to file — Nominatim called once per city ever.
