# 2026-04-08

## What was completed
- **DAT origin auto-fill timing fix**: 3-pass setTimeout chain (pass 1 immediate, pass 2 at +1.5s, pass 3 at +2.5s with full event sequence, blur at +3s) to fight Angular form re-initialization. Uses nativeInputValueSetter to bypass Angular value interception.
- **Multidrop threshold update**: Changed from rejecting 2+ stops to 5+ stops. Numeric regex `/\b([5-9]|\d{2,})\s*stops?\b/i` and written-out words (five through ten). 2-4 stops now acceptable.
- **Reload rejection**: Added "reload", "re-load", "reloading" to comments scanner rejection keywords alongside multidrop and drop-trailer.
- **Commodity miles mismatch detection**: During collection, reads `NNN miles` pattern from full row text. If commodity miles > 1.5x DAT trip miles → RELOAD exclusion. Catches reload loads where the commodity field (e.g. "poultry 1639 miles") reveals a multi-leg total.
- **Full row text scan**: Both scrapers (scrapeRows + scrollAndCollect) now scan `(rowCells || row).textContent.toLowerCase()` for reload/multidrop/drop-trailer keywords. Catches text in commodity, comments preview, or any visible field without opening panels.
- **Click-to-scroll for virtualized list**: Replaced pollForDatRow (10 attempts × 200ms in current DOM only) with scrollAndFindRow (check current DOM → reset to top → scroll in 2x-viewport increments up to 15 steps). Automatically finds rows that have been scrolled out of the virtual DOM. Error message changed from "Load not visible — scroll DAT list and try again" to "Row not found — load may have expired".
- **Destination blacklist added then reverted**: Batesville/Henderson/Pageland blacklist was added then removed per user request.
- **Panel scan approaches tried and reverted**: Two attempts at opening panels during collection to read full comments (top 40 by RPM, click each, wait 1000ms, read COMMENTS section). Both reverted — too slow and unreliable. Final approach uses row text scan instead.
- **PDF parsing audit**: Read-only audit of all PDF/OCR/ratecon infrastructure. Found 7 relevant files. Key finding: no dedicated rate con PDF parser exists. Closest: profit-engine has PDF→Vision OCR, load-optimizer has Tesseract+GPT-4o-mini.

## What's broken / in progress
- **Reload detection from panel comments**: "reload" keyword only appears in the expanded panel COMMENTS section, not in the collapsed row text. The row text scan catches commodity miles mismatch but cannot catch the literal word "reload" in comments without opening panels. Commodity miles detection is the current workaround.
- **Company name false positives**: Row text scan could reject loads from brokers with "reload" in their company name (e.g. "Reload Freight"). No known cases currently.
- **DAT auto-fill flaky**: 3-pass approach works most of the time but Angular may still override on slow connections or heavy page loads.

## Decisions made
- **No panel opens during collection.** Opening 40 panels adds 40+ seconds and is unreliable. Row text scan + commodity miles mismatch is the accepted approach.
- **Multidrop threshold: 5+ stops.** Loads with 2-4 stops are now acceptable per user guidance.
- **Destination blacklists rejected as a pattern.** Too fragile — reload loads change destinations. Commodity miles mismatch is a better heuristic.
- **scrollAndFindRow replaces pollForDatRow.** Automatic virtual scroll search is more reliable than asking the user to manually scroll.
